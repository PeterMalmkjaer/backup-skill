# backup-skill

A Claude Cowork skill for creating reliable project backups — so you can always return to a known good state.

> **Beta — v0.1**  
> Universal: works for any file-based project (code, LaTeX, documents, data, research).

## Core principle

A backup is only useful if you can **find it**, **understand it**, and **restore from it**.

## What it covers

- When to take a backup (before risky operations, not after)
- Naming convention: `BACKUP_YYYYMMDD_vN_label/`
- What to include (and what to skip)
- Where to store backups (two locations minimum)
- Restore procedure (verify before switching, keep broken state)
- Quick bash scripts for Mac/Linux
- Common mistakes and how to avoid them
